Microservices Design Patterns

Microservices Design Patterns

These are software patterns that create reusable, autonomous services

1. Externalized Configuration Configuration settings should be externalized from the codebase, allowing services to be more configurable without requiring redeployment. This can be achieved using tools like ArgoCD manifests, Spring Cloud Config.
2. API Gateway single entry point for clients to access multiple microservices. It can handle routing, authentication, rate limiting, and other cross-cutting concerns.
3. Event-Driven Architecture Microservices can communicate asynchronously using events. Eg: Event Sourcing and CQRS (Command Query Responsibility Segregation)
4. Saga Pattern To maintain data consistency, break the transaction into a series of smaller, compensating transactions
5. CQRS A separate service, model, and database for insert operations in the database. It also has a separate service, model, and database for query data.
